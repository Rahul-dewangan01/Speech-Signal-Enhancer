Project Overview

Speech enhancement is the task of recovering a clean speech signal s(n) from a degraded observation y(n). Real-world speech is corrupted by background noise, reverberation, channel distortion, and interfering speakers. Our project implements and compares seven enhancement algorithms spanning two major paradigms:

⚙️

Adaptive / Classical Methods

  • LMS – Least Mean Squares
  • NLMS – Normalised LMS
  • RLS – Recursive Least Squares
  • Wiener Filter – Decision-directed
  • Spectral Subtraction – Boll 1979

Work in time or frequency domain; exploit statistical models of noise and speech.

🧠

Neural / Deep Learning Methods

  • BLSTM – Bidirectional LSTM mask estimator
  • CRN – Convolutional Recurrent Network

Learn a mapping from noisy STFT features to an Ideal Ratio Mask (IRM) via supervised training on paired data.

Course context: This project is part of an Adaptive Signal Processing course. The neural methods are included to show how classical ideas (Wiener filtering, optimal estimation) motivated deep-learning architectures, and how both can be evaluated with the same metrics.

Noisy Speech Signal Model

Additive Noise Model

The most common model assumes the observed (microphone) signal is an additive mixture:

Time-domain additive model
y(n) = s(n) + d(n) y(n) : observed noisy speech s(n) : clean speech (desired signal) d(n) : additive noise (stationary or non-stationary) n : discrete time index

Signal-to-Noise Ratio (SNR)

SNR = 10·log₁₀ ( P_s / P_d ) [dB] P_s = E[s²(n)] = signal power P_d = E[d²(n)] = noise power

STFT Domain Representation

Most modern methods operate in the Short-Time Fourier Transform (STFT) domain:

Y(k, l) = S(k, l) + D(k, l) k : frequency bin index (0 … N/2) l : time frame index STFT: X(k,l) = Σ_{n=0}^{N-1} x(n + l·H) · w(n) · e^{-j2πkn/N} w(n) : analysis window (Hann) N : FFT size (512) H : hop size (128)

Adaptive Filtering — Core Theory

What is an Adaptive Filter?

An adaptive filter is a digital filter whose coefficients are automatically adjusted by an algorithm to minimise a cost function. Unlike fixed filters, they track non-stationary signals in real time.

┌───────────────────────────────────────────────────────────┐ │ ADAPTIVE NOISE CANCELLER │ │ │ │ Primary mic: y(n) = s(n) + d₁(n) ────────────┐ │ │ ▼ (+) │ │ Reference mic: x(n) ≈ d₀(n) ──► [W(z)] ──► ŷ(n) ──► e(n)│ │ ▲ │ │ │ └─── Weight Update ┘ │ │ (LMS/RLS) │ │ │ │ e(n) = y(n) − ŷ(n) → enhanced speech output │ └───────────────────────────────────────────────────────────┘

The Wiener–Hopf Equation (Optimal Filter)

The Wiener–Hopf equation gives the optimal (minimum MSE) filter weights for stationary inputs:

R_xx · w_opt = r_dx R_xx : (M×M) autocorrelation matrix of input x(n) r_dx : (M×1) cross-correlation vector of desired d(n) and x(n) w_opt: optimal tap-weight vector Minimum MSE (MMSE): ξ_min = σ_d² − r_dx^T · w_opt = σ_d² − r_dx^T · R_xx^{-1} · r_dx
Key insight: Adaptive algorithms (LMS, RLS) are iterative methods that converge toward w_opt without explicitly inverting R_xx. LMS uses gradient descent; RLS uses the matrix inversion lemma for exact updates.

Error Surface and Convergence

The MSE error surface as a function of the weight vector is a paraboloid (bowl-shape). The minimum is ξ_min at w_opt. Adaptive algorithms perform gradient descent on this surface.

MSE as a function of weights: ξ(w) = σ_d² − 2·w^T·r_dx + w^T·R_xx·w Gradient: ∇_w ξ = −2·r_dx + 2·R_xx·w At optimum: ∇_w ξ = 0 → R_xx·w = r_dx (Wiener-Hopf)

LMS, NLMS & RLS Algorithms

4.1 LMS – Least Mean Squares

The LMS algorithm uses an instantaneous gradient estimate (replaces true gradient with one-sample estimate):

LMS Update Rule
Filter output: y(n) = w^T(n) · x(n) Error: e(n) = d(n) − y(n) Weight update: w(n+1) = w(n) + 2μ·e(n)·x(n) μ : step size (learning rate) Stability condition: 0 < μ < 1 / (M · λ_max) where λ_max = largest eigenvalue of R_xx ≈ M·σ_x²

✅ Advantages of LMS

  • Extremely simple: O(M) per sample
  • No matrix operations needed
  • Robust to quantisation errors
  • Works well for stationary noise

❌ Disadvantages of LMS

  • Slow convergence for correlated inputs
  • Step size μ hard to tune
  • Performance degrades for non-stationary noise
  • Gradient noise (misadjustment)

The misadjustment (excess MSE / minimum MSE) for LMS is:

M_adj ≈ μ · tr(R_xx) = μ · M · σ_x² (for small μ) Trade-off: Large μ → fast convergence, high misadjustment Small μ → slow convergence, low misadjustment (steady state)

4.2 NLMS – Normalised LMS

Normalises the step size by the instantaneous input power, giving a variable effective step size that is independent of input power:

NLMS Update Rule
w(n+1) = w(n) + [μ / (ε + ||x(n)||²)] · e(n) · x(n) Effective step: μ_eff(n) = μ / (ε + ||x(n)||²) ε : regularisation (prevents divide-by-zero, typ. 10⁻⁸) Stability: 0 < μ < 2 (for any input power) Optimal: μ = 1 (fastest convergence)
NLMS vs LMS: NLMS achieves the same steady-state MSE as LMS but converges faster when the input has highly variable power (e.g., speech has 40 dB dynamic range). It is the go-to adaptive filter for most practical speech cancellation systems.

4.3 RLS – Recursive Least Squares

Minimises an exponentially-weighted sum of squared errors over the entire history, using the exact second-order statistics:

RLS Cost Function
J(n) = Σ_{i=0}^{n} λ^(n−i) · |e(i)|² λ : forgetting factor (0 < λ ≤ 1) λ=1 → infinite memory (stationary) λ<1 → exponential forgetting (non-stationary)
RLS Recursive Update (via Matrix Inversion Lemma)
k(n) = λ⁻¹ P(n−1) x(n) / [1 + λ⁻¹ x^T(n) P(n−1) x(n)] ← Kalman gain e(n) = d(n) − w^T(n−1) x(n) ← a priori error w(n) = w(n−1) + k(n) · e(n) ← weight update P(n) = λ⁻¹ [P(n−1) − k(n) x^T(n) P(n−1)] ← covariance update Initialisation: w(0) = 0, P(0) = δ⁻¹ I (δ small → fast initial adapt) Complexity: O(M²) per sample

Wiener Filter & Spectral Subtraction

5.1 Frequency-Domain Wiener Filter

In the STFT domain, the optimal Wiener gain for each (frequency bin k, frame l) is:

H_W(k,l) = P_ss(k,l) / [P_ss(k,l) + P_dd(k,l)] = SNR_post(k,l) / [1 + SNR_post(k,l)] P_ss : speech power spectral density P_dd : noise power spectral density Enhanced STFT: Ŝ(k,l) = H_W(k,l) · Y(k,l)

Decision-Directed SNR Estimation (Ephraim–Malah 1984)

Since P_ss is unknown, we estimate the prior SNR using the decision-directed approach:

ξ̂(k,l) = α · |Ŝ(k,l−1)|² / P̂_dd(k,l) + (1−α) · max[γ(k,l)−1, 0] γ(k,l) = |Y(k,l)|² / P̂_dd(k,l) ← posterior SNR ξ̂ : prior SNR estimate α : smoothing constant (typ. 0.92–0.98) Wiener gain becomes: H(k,l) = ξ̂(k,l) / [1 + ξ̂(k,l)]

5.2 Spectral Subtraction (Boll 1979)

Simpler approach: subtract an estimate of the noise magnitude spectrum:

|Ŝ(k,l)| = max( |Y(k,l)| − α_over · |D̂(k,l)| , β·|Y(k,l)| ) α_over : over-subtraction factor (>1, reduces musical noise) typ. 1.0–2.0 β : spectral floor (prevents negative spectra) typ. 0.01–0.05 Phase: φ_Ŝ(k,l) = φ_Y(k,l) (original noisy phase) "Musical noise" artefact: isolated spectral peaks after subtraction → reduced by over-subtraction or non-linear post-filtering
Musical noise is the main artefact of spectral subtraction. It arises because the noise spectrum estimate is noisy itself, creating sporadic residual spikes. The Wiener filter reduces this by smoothing the gain function using the decision-directed approach.

Neural Speech Enhancement

6.1 Masking-Based Enhancement

Neural methods estimate a Time-Frequency mask M(k,l) applied to the noisy magnitude:

|Ŝ(k,l)| = M(k,l) · |Y(k,l)| Ideal Ratio Mask (IRM): M_IRM(k,l) = P_ss(k,l) / [P_ss(k,l) + P_dd(k,l)] ← same as Wiener gain! Ideal Binary Mask (IBM): M_IBM(k,l) = 1 if SNR_local(k,l) > threshold = 0 otherwise
Key connection: The IRM is mathematically identical to the optimal Wiener filter gain. Deep neural networks learn to estimate the IRM from the noisy input, replacing hand-crafted statistical models with learned ones.

6.2 BLSTM Architecture

Input: log-magnitude spectrogram (B, T, 257) │ ├─ LayerNorm(257) │ ├─ BiLSTM Layer 1 [hidden=256 each direction → 512] ├─ BiLSTM Layer 2 [hidden=256 each direction → 512] └─ BiLSTM Layer 3 [hidden=256 each direction → 512] │ ├─ FC(512 → 256) + ReLU + Dropout(0.2) └─ FC(256 → 257) + Sigmoid │ IRM Mask (B, T, 257) ∈ (0,1) │ Enhanced = Mask ⊙ |Y| (element-wise multiply)

The BiLSTM processes each time frame with context from both past and future frames, which is important for non-causal enhancement (offline processing).

6.3 CRN – Convolutional Recurrent Network

Input spectrogram (B, 1, T, F=257) │ ┌────────────────── ENCODER ──────────────────┐ │ Conv2D(1→16, stride_F=2) → BN → ELU e1│ (B,16,T,128) │ Conv2D(16→32, stride_F=2) → BN → ELU e2│ (B,32,T,64) │ Conv2D(32→64, stride_F=2) → BN → ELU e3│ (B,64,T,32) │ Conv2D(64→128,stride_F=2) → BN → ELU e4│ (B,128,T,16) └─────────────────────────────────────────────┘ │ reshape (B, T, 128×16) ┌────────────────── BLSTM BOTTLE ─────────────┐ │ BiLSTM(2048 → 128+128) × 2 layers │ │ Linear(256 → 2048) │ └─────────────────────────────────────────────┘ │ reshape (B, 128, T, 16) ┌────────────────── DECODER ──────────────────┐ │ DeConv(128+128→64) + skip(e4) → BN → ELU │ skip connections │ DeConv(64+64→32) + skip(e3) → BN → ELU │ from encoder │ DeConv(32+32→16) + skip(e2) → BN → ELU │ (U-Net style) │ DeConv(16+16→1) + skip(e1) → BN → ELU │ └─────────────────────────────────────────────┘ │ Sigmoid IRM Mask (B, 1, T, 257)

Evaluation Metrics

7.1 SNR & Segmental SNR

Global SNR = 10·log₁₀ ( ||s||² / ||s − ŝ||² ) [dB] Segmental SNR = (1/L) Σ_l 10·log₁₀ ( ||s_l||² / ||s_l − ŝ_l||² ) Clipped to [−10, 35] dB per frame to avoid outlier dominance. SSNR correlates better with perceptual quality than global SNR.

7.2 PESQ – Perceptual Evaluation of Speech Quality

ITU-T P.862 standard. Computes perceptual disturbance using a model of the human auditory system.

Range: −0.5 to 4.5 (MOS-LQO scale) 4.5 = imperceptible degradation 1.0 = bad quality Wideband (P.862.2): uses 50–7000 Hz, better for 16 kHz speech.
Excellent (>4.0)Acceptable (>3.0)Poor (<2.0)

7.3 STOI – Short-Time Objective Intelligibility

Range: 0 to 1 > 0.9 : Highly intelligible 0.7–0.9: Acceptable < 0.7 : Difficult to understand Computes correlation between clean and enhanced short-time spectral envelopes in one-third octave bands.

7.4 Log-Spectral Distortion (LSD)

LSD = (1/T) Σ_t √[ (1/K) Σ_k (log P_s(t,k) − log P_ŝ(t,k))² ] Smaller is better. Measures spectral shape distortion in log domain.

Interview Q&A — All Questions Answered

Click any question to expand the full answer. These cover every topic a course instructor or interviewer would ask about this project.

🔬 Fundamentals & Theory

Q1. What is the Wiener–Hopf equation and why is it important?
Math

The Wiener–Hopf equation R_xx · w_opt = r_dx gives the closed-form solution for the optimal linear filter weights that minimise the mean-squared error between the filter output and a desired signal.

R_xx · w_opt = r_dx → w_opt = R_xx⁻¹ · r_dx

It is important because: (1) it defines the performance bound — no linear time-invariant filter can do better; (2) all adaptive algorithms (LMS, RLS) converge towards this solution; (3) the frequency-domain Wiener filter is derived directly from it and is used in our project's Wiener enhancement module.

The minimum achievable MSE is: ξ_min = σ_d² − r_dx^T · R_xx⁻¹ · r_dx

Q2. Derive the LMS weight update rule from the steepest-descent algorithm.
Math

Step 1: The steepest-descent algorithm updates weights in the direction of the negative gradient:

w(n+1) = w(n) − μ · ∇_w ξ(n) ∇_w ξ = −2·r_dx + 2·R_xx·w(n) (exact gradient)

Step 2: LMS replaces the true gradient with its instantaneous estimate using a single data sample:

ê[∇_w ξ] = −2·e(n)·x(n) (where e(n)=d(n)−y(n))

Step 3: Substituting gives the LMS rule:

w(n+1) = w(n) + 2μ·e(n)·x(n)

The instantaneous gradient is an unbiased estimate of the true gradient: E[−2e(n)x(n)] = −2r_dx + 2R_xx·w(n).

Q3. What is the convergence condition for the LMS algorithm? How does it affect speech enhancement?
Theory

For LMS to converge in the mean-square sense, the step size must satisfy:

0 < μ < 1 / (M · λ_max) where λ_max = largest eigenvalue of R_xx ≈ M · σ_x² (for white input)

In speech enhancement: Speech has a high dynamic range (~40 dB). When speech is loud, σ_x² is large, requiring a very small μ to maintain stability. This makes LMS slow to converge during high-power frames. NLMS solves this by dividing by ||x||² at each step.

Practical rule: Set μ ≈ 0.01 / (M · σ_x²) as a starting point. If the signal is normalised to unit variance, use μ ≈ 0.01 for M=32.

Q4. Why does RLS converge faster than LMS? What is the trade-off?
Comparison

Why faster: RLS minimises the exact exponentially-weighted least-squares cost over all past samples. It estimates and inverts the input correlation matrix R_xx recursively using the Matrix Inversion Lemma. This gives it "second-order" information (curvature of the error surface) while LMS only uses first-order (gradient direction).

RLS converges in approximately 2M iterations (M = filter order), while LMS needs O(λ_max/λ_min · M) iterations, where the eigenvalue spread of R_xx can be very large for correlated inputs.

Trade-off:

  • Complexity: RLS is O(M²) per sample; LMS is O(M). For M=32, RLS needs ~32× more multiplications.
  • Numerical stability: P(n) can become non-positive-definite due to rounding errors. Square-root variants (QR-RLS) fix this.
  • Memory: RLS stores the M×M matrix P(n); LMS stores only the M-vector w(n).
Q5. What is misadjustment? How does it differ for LMS vs NLMS?
Theory

Misadjustment M_adj is the ratio of excess MSE (due to weight noise from gradient estimation) to the minimum MSE:

LMS: M_adj ≈ μ · tr(R_xx) = μ · M · σ_x² NLMS: M_adj ≈ μ / (2 − μ) (independent of input power)

Key difference: LMS misadjustment scales with input power σ_x². For speech, this means the misadjustment varies wildly frame-to-frame (speech vs silence). NLMS misadjustment is constant regardless of input power, making it more predictable and easier to design.

Q6. What is "musical noise" in spectral subtraction and how do you reduce it?
Theory

Musical noise arises because the noise spectrum estimate is itself noisy. When we subtract it, some frequency bins get a large positive residual (spectral peak) while others go near-zero. The result is a random pattern of tone-like artefacts that changes frame-to-frame — perceived as a "musical" or "twittering" background.

Reduction methods:

  1. Over-subtraction (α > 1): subtracts more noise than estimated, pushing most bins to the spectral floor. Reduces tonal artefacts but introduces some signal distortion.
  2. Spectral flooring (β factor): prevents bins from going to zero, creating a more uniform noise floor.
  3. Multi-band spectral subtraction: estimate noise per sub-band with different α values.
  4. Non-linear post-filtering: apply a smooth gain function instead of hard subtraction.
  5. Wiener filter (decision-directed): temporal smoothing of the gain via α parameter nearly eliminates musical noise.

⚙️ Adaptive Filter Implementation

Q7. How is the adaptive filter used as a noise canceller in this project?
Implementation

We use the Adaptive Noise Cancellation (ANC) architecture:

  • Primary input y(n) = s(n) + d₁(n): speech corrupted by noise from a near-field microphone.
  • Reference input x(n) ≈ d₀(n): a reference microphone close to the noise source, with little or no speech.

The filter adapts to estimate d₁(n) from x(n). The error e(n) = y(n) − ŷ(n) is the noise-cancelled output (enhanced speech).

In our synthetic demo, the reference is the original noise signal (ideal case). In real deployments, the reference is an additional microphone or a delayed version of the primary.

Q8. What is the forgetting factor λ in RLS and how should it be chosen?
Theory
Effective memory length ≈ 1 / (1−λ) samples λ = 0.99 → memory ≈ 100 samples (50 ms at 2 kHz update) λ = 0.95 → memory ≈ 20 samples λ = 1.00 → infinite memory (fully stationary assumption)

Choice guidelines:

  • Stationary noise (office HVAC): λ = 0.99–0.999
  • Slowly varying (car noise): λ = 0.99
  • Fast-varying (babble): λ = 0.95–0.98
  • Rule of thumb: effective memory should span one or two correlation lengths of the noise process.
Q9. What is the difference between time-domain and frequency-domain adaptive filtering?
Comparison

Time-domain: LMS/NLMS/RLS operate directly on the sample stream. Good for low-latency, causal processing. Computational cost O(M) per sample for LMS. For long filters (M > 256), convergence is slow due to eigenvalue spread of R_xx.

Frequency-domain (block LMS, FLMS): Process a block of N samples via FFT, apply updates in frequency domain. Complexity O(N log N) for N-sample block. Converges faster for coloured inputs because frequency-domain updates effectively normalise each frequency bin independently (like per-bin NLMS).

Our Wiener filter and spectral subtraction are frequency-domain methods that update the noise PSD estimate per frame.

🧠 Neural Methods

Q10. Why is the IRM mathematically equivalent to the Wiener filter gain?
Math

Both minimise the MSE between estimated and clean speech in the magnitude domain:

Wiener gain: H(k,l) = P_ss / (P_ss + P_nn) IRM: M(k,l) = P_ss / (P_ss + P_nn) ← identical!

The key insight: the neural network learns to estimate P_ss/(P_ss+P_nn) from the noisy observation, replacing the statistical noise/speech power models used in classical Wiener filtering with a data-driven estimator. The IRM is the theoretical upper bound on mask-based enhancement.

Q11. Why use a Bidirectional LSTM for speech enhancement?
Theory

Why LSTM over simple RNN? Speech has long-term temporal dependencies (pitch periods, phoneme duration, prosody). LSTMs handle these via gated memory cells (input, forget, output gates), avoiding the vanishing gradient problem in long sequences.

Why bidirectional? Enhancement is typically done offline (not real-time), so we can use future context. BiLSTM processes each frame with context from both past AND future frames, significantly improving mask estimation accuracy (typically +0.5–1 dB SNR vs unidirectional).

Gate equations:

f_t = σ(W_f·[h_{t-1}, x_t] + b_f) ← forget gate i_t = σ(W_i·[h_{t-1}, x_t] + b_i) ← input gate o_t = σ(W_o·[h_{t-1}, x_t] + b_o) ← output gate c_t = f_t⊙c_{t-1} + i_t⊙tanh(W_c·[h_{t-1},x_t]+b_c) ← cell state h_t = o_t ⊙ tanh(c_t) ← hidden state
Q12. What is the advantage of CRN (Convolutional Recurrent Network) over BLSTM alone?
Comparison
  • Frequency downsampling: The CNN encoder reduces the frequency dimension by 16×, so the LSTM sees a compact representation. This dramatically reduces LSTM complexity (from 257 to ~16 features) while preserving multi-scale spectral patterns.
  • Skip connections (U-Net style): High-resolution spectral details are preserved via encoder-decoder skip connections, avoiding information bottleneck.
  • 2D convolution: Captures local time-frequency patterns that 1D temporal models miss.
  • Trade-off: CRN has more parameters and is harder to train. BLSTM converges more reliably with less data.
Q13. What loss function is used and why combine mask loss with magnitude loss?
Implementation
L = 0.7 · MSE(M_pred, M_IRM) + 0.3 · L1(M_pred⊙|Y|, |S|)

Mask loss (MSE): Directly supervises the mask towards the IRM. Stable to optimise, gives clear gradient signal about mask accuracy.

Magnitude loss (L1): Supervises the final enhanced magnitude directly. Catches cases where the mask is wrong but the product M·|Y| still distorts the speech. L1 is preferred over L2 for magnitude because L1 is more robust to outliers (large spectral peaks in voiced speech).

Why combine? The mask loss alone can lead to over-suppression at very noisy bins. The magnitude reconstruction loss anchors the network to produce audibly correct output.

💻 Implementation & Practical Questions

Q14. How do you estimate the noise PSD in real-time (without a reference microphone)?
Implementation

Common single-microphone noise estimation methods:

  1. Initial silence estimation: Assume the first 50–100 ms is noise-only (VAD-based init). Works if there is always a noise-only period at start.
  2. Minimum statistics (Martin 2001): Track the minimum power in each frequency bin over a window. The minimum is a biased estimate of the noise floor (bias correction applied).
  3. MMSE-based recursive update: If current frame power is less than λ×previous noise PSD estimate, update noise estimate. Used in our Wiener filter implementation.
  4. VAD-gated estimation: Update noise PSD only during detected silence frames. VAD uses energy + zero-crossing rate thresholds.

Our implementation uses a combination of (1) for initialisation and (3) for tracking.

Q15. What are the STFT parameters chosen in this project and why?
Implementation
N_FFT = 512 → frame length = 32 ms at 16 kHz HOP = 128 → hop length = 8 ms (75% overlap) Window = Hann (periodic) F = 257 bins (0 to 8 kHz)

Rationale:

  • 32 ms frame: Balances frequency resolution (31 Hz per bin) with temporal resolution. Speech phonemes last ~50–300 ms, so this captures short-time stationarity.
  • 75% overlap: Ensures smooth reconstruction via overlap-add, reduces STFT artefacts, and provides more training frames per utterance.
  • Hann window: Good side-lobe suppression (−31 dB first side-lobe), enables perfect reconstruction with 75% overlap.
Q16. How do you handle variable-length sequences in the neural network training?
Implementation

We use two strategies:

  1. Fixed-length cropping: During training, randomly crop all utterances to a fixed duration (4 seconds). This gives uniform batch sizes and allows standard batching.
  2. Zero-padding + collate_fn: Our custom collate_fn pads shorter sequences in a batch to the longest sequence length with zeros. The loss is computed over all frames (including padded ones, which contribute near-zero loss since the masked output is also near-zero).

For inference on arbitrary-length signals, the NeuralEnhancer class processes the full signal without chunking (LSTM processes the entire sequence as one batch).

Q17. Why clip gradient norms during LSTM training?
Implementation

LSTMs are prone to exploding gradients when sequences are long and the loss gradient flows back through many time steps. The product of many Jacobian matrices can grow exponentially.

if ||∇|| > threshold: ∇ ← ∇ × (threshold / ||∇||) threshold = 5.0 in our project

Without clipping, a single "bad" batch can produce a gradient so large it catastrophically updates the weights. Gradient clipping maintains training stability while still allowing fast learning. Note: this is distinct from gradient vanishing, which LSTMs solve via gating.

📊 Evaluation & Results

Q18. Why is PESQ a better metric than SNR for speech quality evaluation?
Theory

SNR limitations: SNR treats all frequency bins equally and all error types equally. A 10 dB improvement in high-frequency noise (less audible) looks the same as a 10 dB improvement in speech-frequency noise (very audible). SNR does not model auditory masking.

PESQ advantages:

  • Uses a perceptual model of the human auditory system (equal-loudness curves, auditory masking)
  • Separately models additive distortion and time-frequency warping
  • Correlates well (r ≈ 0.93) with subjective MOS scores in ITU-T tests
  • Sensitive to speech distortion introduced by over-suppression (a common neural model failure mode)

It is possible to increase SNR by over-suppressing the noise while heavily distorting speech — PESQ will penalise this correctly; SNR will not.

Q19. Why do adaptive filters sometimes hurt performance at high SNR?
Theory

At high SNR (>15 dB), the noise level is already low. Adaptive filters introduce:

  • Misadjustment noise: Random weight fluctuations add a noise floor proportional to μ.
  • Speech cancellation: If the reference signal leaks speech, the adaptive filter partially cancels the desired signal. At high SNR, this speech distortion becomes perceptually dominant.
  • Wiener overattenuation: When noise PSD is overestimated (a common error), the gain H is too small, distorting speech.

This is why metrics should be evaluated across a range of input SNRs. The "best" method changes with SNR conditions.

Q20. How would you extend this project to real-time/streaming operation?
Implementation

Adaptive filters (LMS/NLMS/RLS): Already inherently causal and sample-by-sample. No changes needed. Run directly on audio stream.

Wiener / Spectral Subtraction: Process one STFT frame at a time with a short hop (8–16 ms). Already near real-time.

Neural models (BLSTM → LSTM):

  • Replace BiLSTM with unidirectional LSTM (removes future-frame dependency)
  • Process one frame (8 ms hop) at a time, passing hidden states forward
  • Typical latency: one or two frames (16–32 ms) for look-ahead buffer
  • ONNX export → TensorRT for real-time GPU inference; or quantisation for CPU

Tools: PortAudio for audio I/O, ONNX Runtime for efficient inference, ring buffers for overlap-add reconstruction.

Q21. What is the role of the STFT phase in speech reconstruction?
Theory

All our methods enhance only the magnitude of the STFT and retain the noisy phase:

Ŝ(k,l) = M(k,l) · |Y(k,l)| · e^{jφ_Y(k,l)}

This is called the "noisy phase approximation." It works surprisingly well because:

  • The human auditory system is relatively insensitive to phase at high frequencies (>2–3 kHz).
  • At low frequencies, if the magnitude is correct, the phase error mainly causes slight pitch shifts.
  • Phase estimation is extremely difficult (phase is wrapped and noisy).

Phase-aware methods (like complex IRM, deep complex networks) can give another 0.5–1 dB improvement but at significant complexity cost.

Q22. What preprocessing steps are essential before training the neural model?
Implementation
  1. Resampling to common rate (16 kHz): Ensures all features are in the same frequency range.
  2. Peak normalisation: Prevents clipping and ensures consistent input amplitude. We normalise each training clip to peak amplitude 1.0.
  3. Log-magnitude input: Speech magnitude spectra span a huge dynamic range (~60 dB). Log-magnitude compresses this to a more learnable range and matches the logarithmic loudness perception of the ear.
  4. Multi-condition training: Mix clean speech with multiple noise types at multiple SNRs (0, 5, 10, 15 dB). This makes the model robust to unseen conditions.
  5. Synchronised STFT: Clean and noisy signals must have identical STFT parameters so the mask target is computed correctly.
Q23. How does the Ephraim–Malah decision-directed approach reduce musical noise?
Theory

Musical noise arises from rapid frame-to-frame fluctuation in the Wiener gain. The decision-directed approach introduces temporal smoothing of the prior SNR estimate:

ξ̂(k,l) = α · |Ŝ(k,l−1)|²/σ²_d(k,l) + (1−α) · max[γ(k,l)−1, 0] First term: "decision" from previous enhanced frame (smooth) Second term: "data-driven" from current observation (noisy) α ≈ 0.98: heavily weight the smooth prior estimate

Because ξ̂ is mostly determined by the previous enhanced frame (which is itself smooth), the resulting Wiener gain H = ξ̂/(1+ξ̂) varies smoothly over time. Rapid gain fluctuations that cause musical noise are suppressed. The trade-off is a small amount of "hangover" — noise can persist briefly after the noise stops, but this is less objectionable than musical noise.

Q25. What are the limitations of this project and future improvements?
Theory

Current limitations:

  • Adaptive filters require a separate reference microphone (dual-microphone setup).
  • Neural models trained on synthetic noise may not generalise to all real-world noise types.
  • Phase is not enhanced (noisy phase approximation).
  • No reverberation handling (only additive noise addressed).
  • Single-channel only (no spatial processing).

Future improvements:

  • Complex-domain neural networks (DCCRN, DPCRN) — handle both magnitude and phase.
  • Diffusion model-based enhancement (SGMSE+, CDiffuSE) — state-of-the-art generative approach.
  • Self-supervised pre-training (wav2vec 2.0 features) — reduce labelled data requirements.
  • Beamforming + neural — combine spatial filtering (multi-mic) with neural post-processing.
  • Real-time streaming — unidirectional LSTM + ONNX quantisation for edge deployment.
